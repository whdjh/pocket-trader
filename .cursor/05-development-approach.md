# 개발 방식

- 단계별로 진행, 한 번에 모든 기능 구현하지 않음
- 점진적으로 리팩토링
- 각 단계는 테스트 가능하고 완전해야 함
- 한 번에 모든 것을 구현하지 않음
- **개발 패턴: UI → 기능 → UI → 기능 (반복)**
- **현재 page.tsx는 실제 개발용, mockpage.tsx는 전체 목업 UI 포함**
- 한 번에 하나씩 점진적으로 기능 구축

## 목업 파일 및 마이그레이션

- **현재 `src/lib/*` 파일들은 목업 구현**
- 이 목업 파일들은 DB로 마이그레이션할 비즈니스 로직 포함
- DB 기반 기능 구축 시 이 목업 구현을 참고로 사용
- 목업 파일: `users.ts`, `session-file.ts`, `trade-history.ts`, `bithumb.ts`, `openai-client.ts`, `fear-greed-index.ts`
- 목업 로직을 DB 기반 구현으로 단계별 마이그레이션
