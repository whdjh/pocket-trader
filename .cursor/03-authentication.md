# 인증

- 로그인 구현은 매우 간단하게
- 세션 기반 인증
- 세션은 DB(Supabase)에 저장
- **로그인: 이름만 입력 (비밀번호 불필요)**
- **User ID: `.env.local`의 `USER_ID` 변수에서 가져옴**
- **User Name: 세션(메모리)에만 저장, 표시용**
- **Users 테이블: id (env에서), name (로그인에서), created_at, updated_at**
- **비밀번호 필드 없음** - 로컬 환경, 가족/친구 사용
- 세션은 개인화된 데이터(거래 기록, 잔고)를 위해 필요
- 각 사용자는 자신의 로컬 인스턴스와 `.env.local` 사용
- 각 사용자 데이터는 환경 변수의 user_id로 구분
